About the projects

Throughout my training as a web developer, I’ve worked on different projects that have helped me gain experience with various technologies. In this portfolio, I’m going to show three of them, explaining a bit about what they are, which technologies I used, and how they were carried out. Each project helped me face different challenges and learn new ways to solve problems, which has been key to my professional growth.

Projects

StockMaster

Mi último proyecto consiste en una web de gestión de productos, en la que los usuarios pueden tener diferentes almacenes con sus respectivos productos. A partir de los datos almacenados, se generan gráficos automáticamente, y gracias a la integración de inteligencia artificial es posible crear cualquier tipo de gráfico personalizado.

Además, cuenta con un sistema de chat con IA que permite hacer preguntas sobre el funcionamiento de la web o consultar información relacionada con los productos, como el stock disponible, el producto más caro, entre otros.

Una de las funciones principales es el escáner, que permite añadir o retirar productos de los almacenes simplemente escaneando su código de barras.

Alicia Estetica

Proyecto realizado de forma grupal, consiste en una web para una clínica estética. Entre sus principales funciones destacan la tienda de productos, donde desde el lado del administrador se pueden ir añadiendo nuevos productos.

También incluye un formulario de reserva que solo permite seleccionar días y horas disponibles. Una vez que otro usuario ha reservado una cita con ese profesional en una fecha y hora determinada, esa franja deja de estar disponible.

Por último, la página principal muestra todos los servicios que ofrece la clínica, organizados de forma clara y accesible para los usuarios.

EcoBuddy

Este proyecto fue realizado en grupo. Se trata de una red social cuyo objetivo es incentivar a las personas a realizar acciones positivas para el medio ambiente.

Los usuarios pueden apuntarse a actividades como recogida de basura, plantación de árboles y muchas otras propuestas por la comunidad. Además, cuenta con un sistema de premios que motiva a seguir participando en estas actividades.

El objetivo final de esta red social es fomentar este tipo de acciones y generar conciencia sobre la importancia de cuidar el medio ambiente.

StockMaster -- Inventory Manager

Fue mi proyecto final para DAW, al que le dediqué bastante tiempo y con el que aprendí muchísimo, ya que utilicé múltiples librerías y APIs, además de profundizar mucho más en tecnologías que ya conocía como Angular y Symfony.

En este proyecto destaca el uso de la IA. Gracias a esto desarrollé un panel de notificaciones inteligente que avisa al usuario sobre cualquier incidencia: desde errores simples como un nombre mal introducido en la base de datos hasta problemas más relevantes como un stock bajo o precios incoherentes. Además, implementé un chatbot con acceso a todos los datos del usuario, capaz de responder preguntas relacionadas con los productos, y un sistema de generación de gráficos por IA. El usuario solo tiene que decir qué gráfico quiere generar, y el sistema lo crea automáticamente a partir de los datos almacenados.

Para este proyecto utilicé las siguientes tecnologías:

  • Symfony: Me encargué del backend, creando una API RESTful que conecta con el frontend y gestiona toda la lógica y conexión con la base de datos.
  • Angular: Todo el frontend está desarrollado en Angular. Utilicé librerías como chart.js, zxing/ngx-scanner para el escáner de códigos de barras, OpenLayers para mapas interactivos, entre otras. También integré la API de DeepSeek para toda la parte de IA.
  • HTML/CSS: Para estructurar y dar estilo a los componentes de la interfaz.
  • Bootstrap: Me permitió crear una web responsive y bien estructurada de forma rápida.
  • AWS: Utilicé los servicios de AWS para desplegar y alojar la aplicación en producción.
  • MySQL: Utilicé los MySQL para crear la base de datos.

Alicia Estética -- Sale of products and reservations

Este fue un proyecto en grupo que realizamos para una clínica estética. El enfoque principal fue crear un frontend muy bien hecho y visualmente atractivo, ya que eso era lo que más buscaba el cliente.

La página cuenta con una sección de tienda, donde los usuarios pueden comprar los distintos productos disponibles. También incluye un apartado de reservas, donde puedes elegir el servicio que quieras con el empleado que prefieras. Además, tiene varias páginas informativas que explican todo lo que ofrece la clínica.

Las tecnologías utilizadas en este proyecto son las siguientes:

  • Symfony: Se utilizó para desarrollar el backend, creando una API RESTful que conecta con la base de datos.
  • Angular: Se encargó del frontend de la aplicación. En este caso, no hicimos uso de librerías externas.
  • HTML/CSS: Para estructurar y dar estilos a toda la interfaz.
  • Bootstrap: Fue fundamental para que la web fuera completamente responsive, algo importante ya que muchos usuarios acceden desde el móvil.
  • Docker: Utilizado para ejecutar y contener la aplicación durante el desarrollo.
  • Git: Herramienta esencial para el trabajo en grupo. Cada uno trabajaba en su rama y fuimos integrando los cambios en main.
  • AWS: Se usó para desplegar la aplicación en la nube.
  • MySQL: Para gestionar todos los datos de productos, clientes y reservas.

EcoBuddy

EcoBuddy fue un proyecto en equipo que consistía en el desarrollo de una red social enfocada en acciones para mejorar el medio ambiente. Los usuarios podían publicar actividades como recogida de basura, plantación de árboles y otras iniciativas sostenibles.

El sistema contaba con un sistema de puntos, que se otorgaban al finalizar cada actividad dependiendo del tiempo invertido y su dificultad. Además, se añadieron trofeos como método de gamificación para motivar la participación continua de los usuarios.

Las tecnologías utilizadas en este proyecto fueron las siguientes:

  • PHP: El desarrollo se hizo completamente en PHP puro, tanto el backend como el frontend, sin uso de frameworks.
  • HTML/CSS: Para estructurar la página y darle estilo de manera personalizada.
  • Git: Cada integrante trabajaba en su propia rama y se iban integrando los cambios al repositorio principal usando main.
  • Docker: Utilizado para montar el entorno de desarrollo y ejecutar la aplicación de forma aislada.
  • MySQL: La base de datos se creo con MySQL.
